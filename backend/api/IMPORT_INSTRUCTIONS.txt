===========================================
HOW TO IMPORT DATABASE SCHEMA TO phpMyAdmin
===========================================

STEP 1: Open phpMyAdmin
------------------------
1. Open your web browser
2. Go to: http://localhost/phpmyadmin
3. Make sure XAMPP MySQL service is running

STEP 2: Create Database (if not exists)
----------------------------------------
1. Click on "New" in the left sidebar
2. Database name: vaidyacare_db
3. Collation: utf8mb4_unicode_ci
4. Click "Create" button

STEP 3: Select Database
------------------------
1. Click on "vaidyacare_db" in the left sidebar to select it

STEP 4: Import SQL File
------------------------
Method 1: Using Import Tab (Recommended)
1. Click on "Import" tab at the top
2. Click "Choose File" button
3. Navigate to: C:\xampp\htdocs\VaidyaCare\api\
4. Select file: import_to_phpmyadmin.sql
5. Click "Go" button at the bottom

Method 2: Using SQL Tab
1. Click on "SQL" tab at the top
2. Open the file: C:\xampp\htdocs\VaidyaCare\api\import_to_phpmyadmin.sql
3. Copy all the SQL code
4. Paste it into the SQL textarea
5. Click "Go" button

STEP 5: Verify Table Created
-----------------------------
1. After import, you should see a success message
2. Check left sidebar - you should see "patients" table under vaidyacare_db
3. Click on "patients" table to view structure
4. You should see columns: user_id, full_name, email, mobile, age, gender, password, patient_id, created_at, updated_at

STEP 6: Test Database Connection
---------------------------------
The PHP API files will automatically connect to this database.
Test by trying to signup a user from your Android app.

===========================================
TROUBLESHOOTING
===========================================

If import fails:
- Make sure vaidyacare_db database is selected
- Check MySQL service is running in XAMPP
- Try Method 2 (SQL tab) instead of Import tab
- Check for error messages in phpMyAdmin

If table already exists:
- The SQL uses "CREATE TABLE IF NOT EXISTS" so it's safe
- It won't overwrite existing data
- If you want to recreate, first drop the table: DROP TABLE patients;
